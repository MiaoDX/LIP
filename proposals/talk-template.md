## 选题标准

好的演讲选题应该同时满足：

1. **有真实经历** — 不是纸上谈兵，有实际踩坑数据
2. **有可复用性** — 听完能自己动手做
3. **有反直觉的点** — "我以为 X，结果是 Y" 比 "X 很好" 更有吸引力
4. **和 OPC 目标对齐** — 讲完能引流到公众号/网站

**选题矩阵**：

| 好选题 | 差选题 |
|--------|--------|
| "AI Agent 宕机 6 小时后我做了什么" | "AI Agent 技术介绍" |
| "两个 AI Agent 怎么协作不打架" | "多 Agent 系统概述" |
| "用 Tailscale 稳定使用 Claude Code" | "VPN 技术详解" |

## 大纲模板（45 分钟演讲）

```
1. 开场钩子 (3 min)
   - 一个问题 / 一个意外 / 一个数据
   - 例："我们的 AI Agent 在群里公开泄露了 API key..."

2. 背景交代 (5 min)
   - 我是谁、在做什么、为什么做这个
   - 简短，不要超过 5 分钟

3. 核心故事 (20 min)
   - 分 3-4 个阶段讲
   - 每个阶段：遇到了什么 → 怎么想的 → 怎么做的 → 结果如何
   - 用具体数据：延迟 80ms、花费 $7/月、宕机 6 小时

4. 可复用的经验 (10 min)
   - 提炼 3-5 条，每条配一个真实案例
   - 不要讲理论，讲"我试过 X，结果 Y，所以现在用 Z"

5. 下一步 + 互动 (7 min)
   - 接下来要做什么
   - Q&A
```

## 素材清单

每次演讲前准备：

- [ ] **架构图**（1-2 张）— ASCII 或 SVG，展示系统全貌
- [ ] **数据表格**（2-3 个）— 对比表、费用表、时间线
- [ ] **截图**（3-5 张）— Slack 对话、终端输出、部署界面
- [ ] **代码片段**（可选）— 关键配置，不超过 20 行
- [ ] **失败案例**（至少 1 个）— 比成功案例更有说服力

## 03-15 Meetup 实战复盘

### 做得好的
- 有真实数据和截图，说服力强
- 三层防护架构的讲解效果好
- 双平台对比（ClawCloud vs Railway）实用

### 可以改进的
- 开场钩子不够强 — 下次用更戏剧化的开头
- 时间控制 — 中间部分拖沓，Q&A 时间被压缩
- 互动环节少 — 可以提前准备 2-3 个引导问题

### 下次尝试
- 开场直接放"宕机 6 小时"的故事，不要从背景开始
- 中间穿插 1-2 个小互动（举手投票、现场 demo）
- 结尾放二维码，引流到公众号

## PPT 制作规范

- **风格**：深色背景 + 亮色文字（终端风格）
- **字体**：中文用思源黑体，英文用 JetBrains Mono
- **每页字数**：≤ 30 字，图表优先
- **工具**：HTML 自包含（放在 `share/presentations/`）
- **配套素材**：图片放同目录，用相对路径引用
