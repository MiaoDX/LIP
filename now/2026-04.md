### 多 Agent 协作主权框架
- 写入 TOOLS.md，核心原则：**扩展搜索空间可外包，收缩解释空间留主线程**
- delegation 五问：每一步问自己是在扩展还是收缩
- 月度 audit 框架：谁在实际决定解释轴、命名、主次排序

### 部署安全加固
- deploy-lip.sh 错误推送事件 → 根目录 index.html 被覆盖
- 加固措施：脚本加 guard、限定写入范围、发布前检测根目录变更
- 预防原则：每个项目部署走自己的 CI，禁止越界

### 模型接入扩展
- GPT-5.4 通过 Codex CLI OAuth 接入 OpenClaw
- 新增 model provider 配置，/model 切换验证通过

## 学到什么

1. **对话即接口**：#watercooler 禁用防循环后，Agent 协作更自然
2. **主权比效率重要**：delegation 时守住解释空间收缩权
3. **部署脚本需要 guard**：自动化越界比手动错误更难发现
4. **模型接入标准化**：OAuth → auth-profiles → provider 配置，可复用
